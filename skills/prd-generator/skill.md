# PRD 产品需求文档生成器

帮助产品经理快速生成以用户体验为核心的产品需求文档。

## 使用场景

- 用户想要为新功能编写 PRD
- 用户需要整理需求思路
- 用户提供了功能想法，需要形成正式文档

## 工作流程

### 第一步：收集信息

如果用户没有提供足够信息，通过 AskUserQuestion 工具询问以下关键问题：

1. **功能名称**：这个功能叫什么？
2. **问题背景**：要解决什么问题？用户当前的痛点是什么？
3. **目标用户**：谁会使用这个功能？他们的特征是什么？
4. **核心场景**：用户在什么情况下会用到？使用频率如何？
5. **成功指标**：如何衡量用户体验是否提升？

### 第三步：输出方式

根据用户偏好选择输出方式：

1. **直接输出**：在对话中展示完整 PRD
2. **保存文件**：保存为 `.md` 文件到指定位置
3. **写入飞书**：如果用户配置了飞书 MCP，可以直接创建飞书文档

询问用户："PRD 已生成，你希望我：(1) 直接展示 (2) 保存为文件 (3) 写入飞书文档？"

## 交互原则

1. **不要一次问太多问题** - 分批次收集信息，先问核心问题
2. **聚焦用户体验** - 始终从用户视角思考问题
3. **迭代完善** - 先出初稿，再根据反馈修改
4. **量化目标** - 尽量用数据定义成功

## 示例对话

**用户**：帮我写个用户评论功能的 PRD

**助手**：好的，我来帮你生成用户评论功能的 PRD。先确认几个关键信息：

1. 这个评论功能用在什么产品/场景？（如：电商商品评论、文章评论等）
2. 用户多久会使用一次评论功能？是高频还是低频场景？
3. 目前用户没有评论功能时是怎么表达反馈的？

[收集信息后生成完整 PRD]

## 快捷用法

用户可以直接提供简述，跳过问答：

```
/prd 电商APP的商品收藏功能，解决用户找不到心仪商品的问题，面向年轻女性用户，高频使用，核心是支持分组管理和降价提醒
```

助手将基于这些信息直接生成 PRD 初稿。
